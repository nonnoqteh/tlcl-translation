# فصل ۱۲ — آشنایی با vi
### Chapter 12 – A Gentle Introduction to vi

---

وقتی اولین بار به لینوکس کار کردم، سریعاً متوجه شدم که دانش ویرایشگر متن یک مهارت حیاتی است. مثل بسیاری از مهارت‌های لینوکس، یادگیری آن در ابتدا دشوار به نظر می‌رسد اما پس از تسلط، فوق‌العاده قدرتمند می‌شود.

دستور این فصل:
- `vi` — ویرایشگر متن

---

## چرا باید vi یاد بگیریم؟
### Why We Should Learn vi

در دنیای لینوکس، ویرایشگرهای گرافیکی زیادی وجود دارند مثل gedit، Kate و... اما vi (یا vim — نسخه‌ی بهبودیافته‌اش) دلایل مهمی برای یادگیری دارد:

- **همیشه و همه‌جا هست:** vi در همه‌ی سیستم‌های شبه‌یونیکس نصب است — از سرورهای کوچک تا ابرکامپیوترها. اگر روی یک سرور بدون رابط گرافیکی کار می‌کنی، vi تنها انتخاب ممکن است.
- **سبک‌وزن است:** روی ضعیف‌ترین سیستم‌ها هم کار می‌کند.
- **پس از تسلط فوق‌العاده سریع است:** کاربران حرفه‌ای بدون برداشتن دست از کیبورد کار می‌کنند.

---

## شروع کار با vi

```bash
vi
```

یا برای باز کردن فایل:

```bash
vi foo.txt
```

### حالت‌های vi
### vi Modes

**مهم‌ترین چیزی که باید بدانی:** vi **چندحالته** (multimodal) است. دو حالت اصلی دارد:

- **حالت فرمان** (Command mode) — حالت پیش‌فرض. در این حالت کلیدها فرمان هستند، نه متن.
- **حالت درج** (Insert mode) — در این حالت می‌توانیم متن تایپ کنیم.

> **بزرگ‌ترین مشکل مبتدیان:** فراموش کردن در کدام حالت هستند! هر وقت گیج شدی، `Esc` را فشار بده تا به حالت فرمان برگردی.

---

## حرکت در vi

در حالت فرمان، این کلیدها مکان‌نما را جابجا می‌کنند:

| کلید | حرکت |
|------|------|
| `l` یا `→` | یک کاراکتر به راست |
| `h` یا `←` | یک کاراکتر به چپ |
| `j` یا `↓` | یک خط پایین |
| `k` یا `↑` | یک خط بالا |
| `0` | رفتن به ابتدای خط |
| `^` | رفتن به اولین کاراکتر غیرفضای خط |
| `$` | رفتن به انتهای خط |
| `w` | رفتن به ابتدای کلمه‌ی بعدی |
| `b` | رفتن به ابتدای کلمه‌ی قبلی |
| `G` | رفتن به آخرین خط فایل |
| `1G` یا `gg` | رفتن به اولین خط فایل |
| `nG` | رفتن به خط n‌ام |
| `Ctrl-f` | یک صفحه پایین |
| `Ctrl-b` | یک صفحه بالا |

---

## درج متن
### Inserting Text

از حالت فرمان برای رفتن به حالت درج:

| کلید | عملکرد |
|------|--------|
| `i` | درج قبل از کاراکتر جاری |
| `a` | درج بعد از کاراکتر جاری |
| `A` | درج در انتهای خط |
| `o` | باز کردن خط جدید زیر خط جاری |
| `O` | باز کردن خط جدید بالای خط جاری |

پس از تایپ متن، با `Esc` به حالت فرمان برگرد.

---

## ذخیره کردن و خارج شدن
### Saving Our Work

در vi دستورات **ex** (با `:`) برای ذخیره و خروج استفاده می‌شوند:

| دستور | عملکرد |
|-------|--------|
| `:w` | ذخیره‌ی فایل |
| `:w filename` | ذخیره با نام جدید |
| `:wq` | ذخیره و خروج |
| `:q` | خروج (فقط اگر تغییری نداده باشیم) |
| `:q!` | خروج اجباری بدون ذخیره |
| `ZZ` | ذخیره و خروج (بدون دونقطه) |

---

## حذف متن
### Deleting Text

| کلید | عملکرد |
|------|--------|
| `x` | حذف کاراکتر جاری |
| `dd` | حذف خط جاری |
| `ndd` | حذف n خط |
| `dw` | حذف از مکان‌نما تا ابتدای کلمه‌ی بعدی |
| `d$` | حذف از مکان‌نما تا انتهای خط |
| `d0` | حذف از مکان‌نما تا ابتدای خط |

---

## برش، کپی و جای‌گذاری
### Cut, Copy and Paste

در vi به جای «cut» از «yank» (کندن) و به جای «copy» از «put» استفاده می‌شود:

| کلید | عملکرد |
|------|--------|
| `yy` | کپی خط جاری |
| `nyy` | کپی n خط |
| `yw` | کپی از مکان‌نما تا ابتدای کلمه‌ی بعدی |
| `y$` | کپی از مکان‌نما تا انتهای خط |
| `p` | جای‌گذاری بعد از خط جاری |
| `P` | جای‌گذاری قبل از خط جاری |

> **نکته:** عملیات `dd` هم **برش** است — متن حذف‌شده در بافر ذخیره می‌شود و با `p` قابل جای‌گذاری است.

## جستجو و جایگزینی
### Search and Replace

### جستجو

در حالت فرمان:

```
/عبارت
```

و Enter. برای رفتن به نتیجه‌ی بعدی: `n` — به قبلی: `N`.

برای جستجوی معکوس (به بالا):

```
?عبارت
```

### جایگزینی سراسری

```
:%s/قدیم/جدید/g
```

- `%` = در تمام فایل
- `s` = جایگزینی (substitute)
- `g` = تمام موارد در هر خط (global)

برای تأیید هر مورد قبل از جایگزینی:

```
:%s/قدیم/جدید/gc
```

با `c`، vi برای هر مورد می‌پرسد:
- `y` — جایگزین کن
- `n` — رد کن
- `a` — همه را جایگزین کن
- `q` — خروج از جایگزینی

## ویرایش چند فایل
### Editing Multiple Files

```bash
vi file1 file2 file3
```

| دستور | عملکرد |
|-------|--------|
| `:n` | رفتن به فایل بعدی |
| `:N` | رفتن به فایل قبلی |
| `:bn` | رفتن به فایل بعدی (در برخی نسخه‌ها) |
| `:buffers` | نمایش فهرست فایل‌های باز |

برای کپی متن از یک فایل به فایل دیگر، ابتدا در فایل اول متن را yank کن، سپس به فایل دوم برو و paste کن.

---

## درج محتوای فایل دیگر
### Inserting an Entire File into Another

```
:r filename
```

این محتوای `filename` را در خط جاری درج می‌کند.

---

## تنظیمات مفید vi

می‌توانیم تنظیمات vi را در دستور `:set` تعریف کنیم. برای دائمی‌کردن، این خطوط را در `~/.vimrc` بنویس:

```
:set number        # نمایش شماره‌ی خط
:set nonumber      # پنهان کردن شماره‌ی خط
:set ignorecase    # جستجوی غیرحساس به بزرگی/کوچکی
:set hlsearch      # هایلایت نتایج جستجو
:set autoindent    # تورفتگی خودکار
```

---

## خلاصه

vi در ابتدا گیج‌کننده است، اما پس از چند ساعت تمرین شروع به منطقی شدن می‌کند. مهم‌ترین چیز: همیشه بدان در کدام حالت هستی. با تمرین مداوم، vi به یکی از قدرتمندترین ابزارهایت تبدیل می‌شود.
