# فصل ۸ — ترفندهای پیشرفته‌ی صفحه‌کلید
### Chapter 8 – Advanced Keyboard Tricks

---

اغلب می‌گویم یونیکس «سیستم‌عاملی است برای کسانی که دوست دارند تایپ کنند.» اما کاربران خط فرمان واقعاً **دوست ندارند** زیاد تایپ کنند! چرا دستورات اسم‌های کوتاهی مثل `cp`، `ls`، `mv` و `rm` دارند؟ دقیقاً به همین دلیل. یکی از هدف‌های اصلی خط فرمان: **کمترین تایپ ممکن**. در این فصل ویژگی‌های bash برای تایپ سریع‌تر را بررسی می‌کنیم.

دستورات جدید:
- `clear` — پاک کردن صفحه
- `history` — نمایش تاریخچه‌ی دستورات

---

## ویرایش خط فرمان
### Command Line Editing

bash از کتابخانه‌ای به نام **Readline** برای ویرایش خط فرمان استفاده می‌کند.

### حرکت مکان‌نما

| کلید | عملکرد |
|------|--------|
| `Ctrl-a` | رفتن به ابتدای خط |
| `Ctrl-e` | رفتن به انتهای خط |
| `Ctrl-f` | یک کاراکتر جلو (مثل جهت راست) |
| `Ctrl-b` | یک کاراکتر عقب (مثل جهت چپ) |
| `Alt-f` | یک کلمه جلو |
| `Alt-b` | یک کلمه عقب |
| `Ctrl-l` | پاک کردن صفحه و رفتن به گوشه‌ی بالا-چپ |

### ویرایش متن

| کلید | عملکرد |
|------|--------|
| `Ctrl-d` | حذف کاراکتر زیر مکان‌نما |
| `Ctrl-t` | جابجایی کاراکتر زیر مکان‌نما با قبلی |
| `Alt-t` | جابجایی کلمه‌ی زیر مکان‌نما با قبلی |
| `Alt-l` | تبدیل از مکان‌نما تا انتهای کلمه به حروف کوچک |
| `Alt-u` | تبدیل از مکان‌نما تا انتهای کلمه به حروف بزرگ |

### برش و جای‌گذاری (Cut & Paste)

در مستندات Readline به جای «cut» از «kill» و به جای «paste» از «yank» استفاده می‌شود. موارد بریده‌شده در حافظه‌ای به نام **kill-ring** ذخیره می‌شوند.

| کلید | عملکرد |
|------|--------|
| `Ctrl-k` | برش از مکان‌نما تا انتهای خط |
| `Ctrl-u` | برش از مکان‌نما تا ابتدای خط |
| `Alt-d` | برش از مکان‌نما تا انتهای کلمه |
| `Alt-Backspace` | برش از مکان‌نما تا ابتدای کلمه |
| `Ctrl-y` | جای‌گذاری از kill-ring |

## تکمیل خودکار (Completion)
### Completion

یکی دیگر از روش‌های کمک شل، **تکمیل خودکار** (Tab completion) است. هنگام تایپ، کلید Tab را فشار دهید:

```bash
ls l
```

(کلید Tab را بزنید)

```bash
ls ls-output.txt
```

شل خط را کامل کرد! اگر چند احتمال وجود داشته باشد، Tab را دوبار بزنید تا گزینه‌ها نمایش داده شوند:

```bash
ls D
```

(Tab)

نتیجه‌ای نمی‌دهد چون هم `Desktop` و هم `Documents` با `D` شروع می‌شوند. با اضافه کردن حرف بعدی:

```bash
ls Do
```

(Tab)

```bash
ls Documents
```

تکمیل خودکار علاوه بر نام مسیر، برای این موارد هم کار می‌کند:
- **متغیرها** (اگر کلمه با `$` شروع شود)
- **نام‌های کاربری** (اگر با `~` شروع شود)
- **دستورها** (اگر اولین کلمه‌ی خط باشد)
- **نام میزبان‌ها** (اگر با `@` شروع شود)

| کلید | عملکرد |
|------|--------|
| `Alt-?` | نمایش فهرست تکمیل‌های ممکن |
| `Alt-*` | وارد کردن همه‌ی تکمیل‌های ممکن |

## استفاده از تاریخچه
### Using History

bash تاریخچه‌ی دستورات را در فایل `~/.bash_history` نگه می‌دارد. به‌طور پیش‌فرض ۱۰۰۰ دستور آخر ذخیره می‌شوند.

برای مشاهده‌ی تاریخچه:

```bash
history | less
```

### جستجو در تاریخچه

برای یافتن دستور خاص:

```bash
history | grep /usr/bin
```

اگر نتیجه‌ای مثل زیر پیدا کردیم:

```
88  ls -l /usr/bin > ls-output.txt
```

می‌توانیم با شماره آن را اجرا کنیم:

```bash
!88
```

**جستجوی تدریجی معکوس:** برای جستجوی تدریجی در تاریخچه، **Ctrl-r** را بزنید:

```
(reverse-i-search)`':
```

شروع به تایپ کنید، مثلاً `/usr/bin`:

```
(reverse-i-search)`/usr/bin': ls -l /usr/bin > ls-output.txt
```

برای اجرا Enter بزنید، یا با **Ctrl-j** آن را به خط جاری کپی کنید.

**کلیدهای مفید برای تاریخچه:**

| کلید | عملکرد |
|------|--------|
| `Ctrl-p` | دستور قبلی (مثل جهت بالا) |
| `Ctrl-n` | دستور بعدی (مثل جهت پایین) |
| `Alt-<` | رفتن به اولین دستور تاریخچه |
| `Alt->` | رفتن به آخرین دستور (خط جاری) |
| `Ctrl-r` | جستجوی تدریجی معکوس |
| `Ctrl-o` | اجرای دستور جاری و رفتن به بعدی |

### بسط تاریخچه (History Expansion)

| دنباله | عملکرد |
|--------|--------|
| `!!` | تکرار آخرین دستور |
| `!شماره` | تکرار دستور شماره‌ی مشخص |
| `!رشته` | تکرار آخرین دستوری که با آن رشته شروع شده |
| `!?رشته` | تکرار آخرین دستوری که شامل آن رشته است |

> **احتیاط:** قبل از استفاده از `!رشته` مطمئن شوید تاریخچه‌ی درستی در ذهن دارید. می‌توانید با `:p` فقط نتیجه را چاپ کنید بدون اینکه اجرا شود:
> ```bash
> !ls:p
> ```

## script — ضبط نشست ترمینال

اکثر توزیع‌های لینوکس دستوری به نام `script` دارند که کل نشست ترمینال را ضبط می‌کند:

```bash
script [نام_فایل]
```

اگر نام فایل ندهید، به‌طور پیش‌فرض `typescript` استفاده می‌شود.
