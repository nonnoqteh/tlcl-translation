# فصل ۱ — شل چیست؟
### Chapter 1 – What Is the Shell?

---

وقتی از «خط فرمان» صحبت می‌کنیم، در واقع از **شل** (shell) حرف می‌زنیم. شل برنامه‌ای است که دستورات تایپ‌شده را دریافت کرده و برای اجرا به سیستم‌عامل تحویل می‌دهد. تقریباً همه‌ی توزیع‌های لینوکس از برنامه‌ای به نام **bash** استفاده می‌کنند که از پروژه‌ی گنو تأمین می‌شود. نام «bash» مخفف «Bourne Again SHell» است — اشاره‌ای به اینکه bash نسخه‌ی بهبودیافته‌ی `sh` است، شل اصلی یونیکس که توسط استیو بورن نوشته شد.

---

## شبیه‌ساز ترمینال
### Terminal Emulators

در محیط گرافیکی (GUI)، برای ارتباط با شل به برنامه‌ای به نام **شبیه‌ساز ترمینال** (terminal emulator) نیاز داریم. اگر در منوی دسکتاپ بگردیم، احتمالاً یکی پیدا می‌کنیم. KDE از `konsole` و GNOME از `gnome-terminal` استفاده می‌کند، هرچند در منو معمولاً فقط «Terminal» نام دارد. ترمینال‌های دیگری هم برای لینوکس وجود دارند، اما همه یک کار می‌کنند: دسترسی به شل.

---

## اولین تایپ‌ها
### Making Your First Keystrokes

بیایید شروع کنیم. ترمینال را باز کنید! پس از باز شدن، چیزی شبیه به این می‌بینید:

```
[me@linuxbox ~]$
```

این را **فراخوان شل** (shell prompt) می‌نامند و هر بار که شل آماده‌ی دریافت دستور است ظاهر می‌شود. معمولاً شامل `نام‌کاربری@نام‌دستگاه`، پوشه‌ی جاری و یک علامت `$` است.

> **نکته:** اگر آخرین کاراکتر فراخوان `#` بود (نه `$`)، ترمینال با دسترسی سوپریوزر (مدیریتی) در حال اجرا است.

بیایید چیزی تایپ کنیم. یک چرت‌وپرت وارد کنید:

```bash
[me@linuxbox ~]$ kaekfjaeifj
```

```
bash: kaekfjaeifj: command not found
[me@linuxbox ~]$
```

چون این دستور معنایی ندارد، شل خطا می‌دهد و دوباره فراخوان را نشان می‌دهد.

## تاریخچه‌ی دستورات
### Command History

اگر کلید **جهت بالا** را فشار دهید، دستور قبلی دوباره ظاهر می‌شود. این قابلیت را **تاریخچه‌ی دستورات** (command history) می‌نامند. اکثر توزیع‌های لینوکس ۱۰۰۰ دستور آخر را به‌طور پیش‌فرض به خاطر می‌سپارند. با کلید **جهت پایین** می‌توانید دستور را پاک کنید.

---

## حرکت مکان‌نما
### Cursor Movement

با کلیدهای جهت چپ و راست می‌توانید مکان‌نما را در هر نقطه‌ای از خط دستور قرار دهید. این ویرایش دستورات را آسان می‌کند.

> **نکته درباره‌ی ماوس:** در ترمینال می‌توانید متن را با نگه‌داشتن دکمه‌ی چپ ماوس انتخاب و با دکمه‌ی وسط جای‌گذاری کنید. **از Ctrl-c و Ctrl-v برای کپی/پیست در ترمینال استفاده نکنید** — آن‌ها در شل معنای متفاوتی دارند.

---

## چند دستور ساده
### Try Some Simple Commands

حالا چند دستور ساده را امتحان کنیم. دستور `date` تاریخ و ساعت جاری را نشان می‌دهد:

```bash
date
```

```
Thu Mar  8 15:09:41 EST 2025
```

دستور `uptime` مدت زمانی که سیستم روشن بوده را نمایش می‌دهد:

```bash
uptime
```

```
15:12:22 up 3 days, 23:40,  7 users,  load average: 0.37, 0.37, 0.64
```

برای دیدن فضای آزاد روی دیسک از `df` استفاده کنید:

```bash
df
```

```
Filesystem           1K-blocks      Used Available Use% Mounted on
/dev/sda2             15115452   5012392   9949716  34% /
/dev/sda5             59631908  26545424  30008432  47% /home
```

برای نمایش حافظه‌ی آزاد از `free` استفاده کنید:

```bash
free
```

```
         total       used       free     shared    buffers     cached
Mem:    513712     503976       9736          0       5312     122916
-/+ buffers/cache: 375748     137964
```

## پایان دادن به نشست ترمینال
### Ending a Terminal Session

می‌توانید نشست ترمینال را با بستن پنجره، تایپ `exit` یا فشردن **Ctrl-d** پایان دهید:

```bash
exit
```

> **ترمینال‌های مجازی:** حتی اگر هیچ ترمینال گرافیکی باز نباشد، چند نشست ترمینال در پشت‌زمینه اجرا می‌شوند. با **Ctrl-Alt-F1** تا **Ctrl-Alt-F6** می‌توانید به آن‌ها دسترسی داشته باشید. برای بازگشت به دسکتاپ گرافیکی **Alt-F7** را فشار دهید.
