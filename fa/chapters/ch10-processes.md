# فصل ۱۰ — فرآیندها
### Chapter 10 – Processes

---

سیستم‌های عامل مدرن معمولاً **چندوظیفه‌ای** (multitasking) هستند — توهمی از اجرای همزمان چند برنامه با جابجایی سریع بین آن‌ها ایجاد می‌کنند. هسته‌ی لینوکس این کار را از طریق **فرآیندها** (processes) مدیریت می‌کند.

دستورات این فصل:
- `ps` — نمایش فرآیندهای جاری
- `top` — نمایش پویای فرآیندها
- `jobs` — فهرست کارهای فعال
- `bg` — ادامه‌ی کار در پس‌زمینه
- `fg` — ادامه‌ی کار در پیش‌زمینه
- `kill` — ارسال سیگنال به فرآیند
- `killall` — ارسال سیگنال به چند فرآیند
- `shutdown` — خاموش یا راه‌اندازی مجدد سیستم

---

## چطور یک فرآیند کار می‌کند؟
### How a Process Works

وقتی سیستم راه‌اندازی می‌شود، هسته یک سری **فرآیند‌های دیمون** (daemon processes) می‌سازد که در پس‌زمینه مشغول به کارند. برنامه‌های کاربری هم فرآیندهای خودشان را می‌سازند.

هسته برای هر فرآیند یک **شناسه‌ی فرآیند** (PID — Process ID) اختصاص می‌دهد. PIDها به ترتیب صعودی تخصیص داده می‌شوند. اولین فرآیند سیستم `init` است با PID 1.

## مشاهده‌ی فرآیندها
### Viewing Processes

### `ps` — عکس لحظه‌ای از فرآیندها

```bash
ps
```

```
  PID TTY          TIME CMD
 5198 pts/1    00:00:00 bash
10129 pts/1    00:00:00 ps
```

فقط فرآیندهای متصل به ترمینال جاری نشان داده می‌شوند. برای دیدن بیشتر:

```bash
ps x
```

```
  PID TTY      STAT   TIME COMMAND
 2799 ?        Ssl    0:00 /usr/libexec/bonobo-activation-server...
 2820 ?        Sl     0:01 /usr/libexec/evolution-data-server-1.10
...
10133 pts/1    R+     0:00 ps x
```

ستون **STAT** (وضعیت فرآیند):

| کد | معنی |
|----|------|
| `R` | در حال اجرا (Running) |
| `S` | خوابیده (Sleeping) — منتظر رویداد |
| `D` | خوابیده‌ی وقفه‌ناپذیر — منتظر I/O |
| `T` | متوقف‌شده (Stopped) |
| `Z` | زامبی — فرآیند مُرده که هنوز پاک نشده |
| `<` | اولویت بالا |
| `N` | اولویت پایین |

برای نمایش همه‌ی فرآیندهای سیستم:

```bash
ps aux
```

```
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root         1  0.0  0.0   2136   644 ?        Ss   Mar05   0:31 init
root         2  0.0  0.0      0     0 ?        S<   Mar05   0:00 [kt/0]
...
me       10134  4.3  0.3  15388  3720 pts/1    R+   16:20   0:00 ps aux
```

### `top` — نمایش پویای فرآیندها

`ps` یک عکس لحظه‌ای می‌گیرد. اما `top` اطلاعات را به صورت **زنده** به‌روز می‌کند (پیش‌فرض: هر ۳ ثانیه):

```bash
top
```

```
top - 14:59:20 up 6:30,  2 users,  load average: 0.07, 0.02, 0.00
Tasks: 109 total,   1 running, 106 sleeping,   0 stopped,   2 zombie
Cpu(s):  0.7%us,  1.0%sy,  0.0%ni, 98.3%id,  0.0%wa
Mem:    319496k total,   314860k used,     4636k free,    19392k buffers
Swap:   875500k total,       148k used,   875352k free,   114192k cached

  PID USER      PR  NI  VIRT  RES  SHR S %CPU %MEM    TIME+  COMMAND
 6244 me        39  19 31752 3124 2188 S  6.3  1.0   0:00.03 trackerd
...
```

**سطر اول:** ساعت سیستم، مدت زمان روشن‌بودن (uptime)، تعداد کاربران، میانگین بار در ۱، ۵ و ۱۵ دقیقه گذشته

**سطر CPU:** درصد اجرا توسط کاربر (`us`)، سیستم (`sy`)، بیکار (`id`)

**کلیدهای `top`:**
| کلید | عملکرد |
|------|--------|
| `h` | راهنما |
| `q` | خروج |
| `M` | مرتب‌سازی بر اساس حافظه |
| `P` | مرتب‌سازی بر اساس CPU |
| `k` | کشتن یک فرآیند (با وارد کردن PID) |

## کنترل فرآیندها
### Controlling Processes

### اجرا در پس‌زمینه (Background)

با افزودن `&` به انتهای دستور، آن را در پس‌زمینه اجرا کنید:

```bash
xlogo &
```

```
[1] 28236
```

شل شماره‌ی **job** (داخل `[]`) و PID را نشان می‌دهد.

```bash
ps
```

```
  PID TTY          TIME CMD
10603 pts/1    00:00:00 bash
28236 pts/1    00:00:00 xlogo
28239 pts/1    00:00:00 ps
```

### `jobs` — فهرست کارهای فعال

```bash
jobs
```

```
[1]+ Running                 xlogo &
```

### `fg` — برگرداندن به پیش‌زمینه

```bash
fg %1
```

`%1` یعنی job شماره‌ی ۱. اگر فقط یک job باشد، شماره لازم نیست.

### متوقف کردن فرآیند (Ctrl-z)

```bash
xlogo
```

(Ctrl-z را فشار دهید)

```
[1]+ Stopped                 xlogo
```

فرآیند **متوقف** شد اما **کشته نشد**. برای ادامه در پس‌زمینه:

```bash
bg %1
```

```
[1]+ xlogo &
```

## سیگنال‌ها و `kill`
### Signals

دستور `kill` سیگنال می‌فرستد، نه لزوماً «می‌کشد»:

```bash
kill [-signal] PID...
```

**سیگنال‌های مهم:**

| شماره | نام | معنی |
|-------|-----|------|
| 1 | `HUP` | قطع‌ارتباط — برای راه‌اندازی مجدد فرآیندهای دیمون |
| 2 | `INT` | وقفه — همان Ctrl-c |
| 9 | `KILL` | کشتن فوری از طرف هسته (قابل بلاک نیست) |
| 15 | `TERM` | پایان دادن — پیش‌فرض `kill` |
| 18 | `CONT` | ادامه — بعد از STOP |
| 19 | `STOP` | توقف (قابل بلاک نیست) |
| 20 | `TSTP` | توقف ترمینال — همان Ctrl-z |

مثال‌ها:

```bash
kill -1 28236       # HUP — راه‌اندازی مجدد
kill -9 28236       # KILL — کشتن اجباری
kill -SIGKILL 28236 # با نام کامل
kill 28236          # TERM — پیش‌فرض
```

---

### `killall` — ارسال سیگنال به چند فرآیند

برای کشتن همه‌ی فرآیندهایی با یک نام:

```bash
killall xlogo
```

## مدیریت اولویت
### Managing Process Priority

لینوکس به هر فرآیند یک **اولویت زمان‌بندی** می‌دهد. بازه از **-20** (بالاترین اولویت) تا **19** (پایین‌ترین) است.

برای شروع فرآیند با اولویت پایین‌تر:

```bash
nice -n 10 cpu-intensive-command
```

برای تغییر اولویت فرآیند در حال اجرا:

```bash
renice 15 -p 28236
```

> **نکته:** کاربر عادی فقط می‌تواند اولویت **پایین‌تر** بدهد. بالا بردن اولویت نیاز به سوپریوزر دارد.

## خاموش کردن سیستم
### Shutting Down the System

```bash
sudo reboot          # راه‌اندازی مجدد
sudo shutdown -h now # خاموش کردن فوری
sudo shutdown -r now # راه‌اندازی مجدد فوری
sudo shutdown -h +10 # خاموش کردن بعد از ۱۰ دقیقه
```

`halt` و `poweroff` هم برای خاموش کردن کار می‌کنند.

## خلاصه

در این فصل ابزارهای پایه برای مشاهده و کنترل فرآیندها را دیدیم. با `ps` و `top` وضعیت سیستم را بررسی می‌کنیم، با `kill` و `killall` فرآیندها را کنترل می‌کنیم، و با `&`، `fg`، `bg` و `jobs` کارها را بین پیش‌زمینه و پس‌زمینه جابجا می‌کنیم.
