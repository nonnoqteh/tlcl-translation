# فصل ۶ — تغییر مسیر (Redirection)
### Chapter 6 – Redirection

---

در این فصل با یکی از جالب‌ترین قابلیت‌های خط فرمان آشنا می‌شویم: **تغییر مسیر ورودی/خروجی** (I/O Redirection). با این قابلیت می‌توانیم ورودی و خروجی دستورها را به فایل‌ها هدایت کنیم و چندین دستور را از طریق **لوله‌کشی** (pipeline) به هم وصل کنیم. دستورات معرفی‌شده:

- `cat` — اتصال فایل‌ها
- `sort` — مرتب‌سازی خطوط متن
- `uniq` — حذف خطوط تکراری
- `grep` — چاپ خطوط منطبق با الگو
- `wc` — شمارش خط، کلمه و بایت
- `head` — نمایش ابتدای فایل
- `tail` — نمایش انتهای فایل
- `tee` — خواندن از ورودی استاندارد و نوشتن به خروجی استاندارد و فایل

---

## ورودی، خروجی و خطای استاندارد
### Standard Input, Output, and Error

بسیاری از برنامه‌ها دو نوع خروجی تولید می‌کنند:
- **نتیجه‌ی برنامه** — داده‌ای که برنامه تولید می‌کند
- **پیام‌های وضعیت و خطا**

در دنیای یونیکس، برنامه‌ها نتایجشان را به یک فایل خاص به نام **خروجی استاندارد** (stdout) می‌فرستند و پیام‌های خطایشان را به **خطای استاندارد** (stderr). به‌طور پیش‌فرض هر دو به صفحه وصل هستند.

علاوه بر آن، برنامه‌ها ورودی را از **ورودی استاندارد** (stdin) دریافت می‌کنند که به‌طور پیش‌فرض به صفحه‌کلید متصل است.

تغییر مسیر I/O به ما اجازه می‌دهد این پیش‌فرض‌ها را عوض کنیم.

---

## تغییر مسیر خروجی استاندارد
### Redirecting Standard Output

برای هدایت خروجی استاندارد به یک فایل از عملگر `>` استفاده می‌کنیم:

```bash
ls -l /usr/bin > ls-output.txt
```

```bash
ls -l ls-output.txt
```

```
-rw-rw-r-- 1 me me 167878 2025-02-01 15:07 ls-output.txt
```

اگر پوشه وجود نداشته باشد:

```bash
ls -l /bin/usr > ls-output.txt
```

```
ls: cannot access /bin/usr: No such file or directory
```

توجه کنید پیام خطا روی صفحه نمایش داده شد، نه در فایل — چون خطا به stderr فرستاده می‌شود. اما فایل:

```bash
ls -l ls-output.txt
```

```
-rw-rw-r-- 1 me me 0 2025-02-01 15:08 ls-output.txt
```

فایل خالی شد! دلیل: عملگر `>` همیشه فایل را از ابتدا بازنویسی می‌کند. می‌توانیم از این رفتار برای خالی کردن یک فایل موجود یا ساختن فایل خالی جدید استفاده کنیم:

```bash
> ls-output.txt
```

برای **اضافه کردن** به انتهای فایل (به جای بازنویسی) از `>>` استفاده کنید:

```bash
ls -l /usr/bin >> ls-output.txt
ls -l /usr/bin >> ls-output.txt
ls -l /usr/bin >> ls-output.txt
ls -l ls-output.txt
```

```
-rw-rw-r-- 1 me me 503634 2025-02-01 15:45 ls-output.txt
```

## دستورات گروهی
### Group Commands

اگر بخواهیم چند دستور را به یک فایل لاگ بفرستیم، می‌توانیم آن‌ها را با آکولاد گروه‌بندی کنیم:

```bash
{ command1; command2; command3; } > logfile.txt
```

---

## تغییر مسیر خطای استاندارد
### Redirecting Standard Error

برای هدایت خطای استاندارد از شماره‌ی توصیف‌گر فایل `2` استفاده می‌کنیم:

```bash
ls -l /bin/usr 2> ls-error.txt
```

برای هدایت هر دو (خروجی و خطا) به یک فایل:

```bash
# روش قدیمی (سازگار با همه نسخه‌ها)
ls -l /bin/usr > ls-output.txt 2>&1

# روش جدید (bash جدید)
ls -l /bin/usr &> ls-output.txt
```

> **ترتیب مهم است!** در روش قدیمی، ابتدا باید خروجی را هدایت کنید و بعد خطا را. اگر ترتیب عوض شود، خطا به صفحه می‌رود.

برای افزودن هر دو به انتهای فایل:

```bash
ls -l /bin/usr &>> ls-output.txt
```

## دور انداختن خروجی ناخواسته
### Disposing of Unwanted Output

گاهی نمی‌خواهیم هیچ خروجی‌ای ببینیم. سیستم یک فایل خاص به نام `/dev/null` دارد که هر چیزی که به آن فرستاده شود، دور انداخته می‌شود:

```bash
ls -l /bin/usr 2> /dev/null
```

> `/dev/null` در فرهنگ یونیکس مفهومی اسطوره‌ای است. وقتی کسی می‌گوید «کامنت‌هایت را به `/dev/null` می‌فرستم» یعنی آن‌ها را نادیده می‌گیرد!

## تغییر مسیر ورودی استاندارد
### Redirecting Standard Input

### `cat` — اتصال فایل‌ها

دستور `cat` یک یا چند فایل را می‌خواند و به خروجی استاندارد کپی می‌کند:

```bash
cat ls-output.txt
```

برای ترکیب فایل‌های تقسیم‌شده:

```bash
cat movie.mpeg.0* > movie.mpeg
```

اگر `cat` را بدون آرگومان اجرا کنید، از ورودی استاندارد (صفحه‌کلید) می‌خواند. این رفتار برای ساختن فایل‌های متنی کوتاه مفید است:

```bash
cat > lazy_dog.txt
The quick brown fox jumps over the lazy dog.
```

(با Ctrl-d پایان می‌دهیم)

برای تغییر مسیر ورودی استاندارد از `<` استفاده می‌کنیم:

```bash
cat < lazy_dog.txt
```

```
The quick brown fox jumps over the lazy dog.
```

## لوله‌کشی (Pipelines)
### Pipelines

قابلیت خواندن از ورودی استاندارد و نوشتن به خروجی استاندارد توسط یک ویژگی شل به نام **لوله‌کشی** (pipeline) استفاده می‌شود. با عملگر `|` (خط عمودی)، خروجی یک دستور به ورودی دستور بعدی وصل می‌شود:

```bash
ls -l /usr/bin | less
```

این بسیار مفید است! می‌توانیم خروجی هر دستوری را صفحه‌به‌صفحه ببینیم.

> **تفاوت `>` و `|`:**
> - `>` یک دستور را به یک **فایل** وصل می‌کند
> - `|` خروجی یک دستور را به **ورودی دستور بعدی** وصل می‌کند
>
> خطرناک: `command1 > command2` — این فایلی به نام `command2` می‌سازد یا آن را بازنویسی می‌کند، نه اینکه دستورات را لوله کند!

## فیلترها
### Filters

لوله‌کشی اغلب برای اجرای عملیات پیچیده روی داده استفاده می‌شود. دستوراتی که به این شکل استفاده می‌شوند **فیلتر** نامیده می‌شوند.

### `sort` — مرتب‌سازی

```bash
ls /bin /usr/bin | sort | less
```

### `uniq` — حذف تکراری‌ها

```bash
ls /bin /usr/bin | sort | uniq | less
```

برای دیدن موارد تکراری (به جای حذفشان):

```bash
ls /bin /usr/bin | sort | uniq -d | less
```

### `wc` — شمارش

`wc` تعداد خط، کلمه و بایت را نشان می‌دهد:

```bash
wc ls-output.txt
```

```
 7902  64566 503634 ls-output.txt
```

برای شمارش تعداد آیتم‌های فهرست:

```bash
ls /bin /usr/bin | sort | uniq | wc -l
```

```
2728
```

### `grep` — جستجوی الگو

`grep` یک ابزار قدرتمند برای یافتن الگوهای متنی است:

```bash
grep pattern [file...]
```

مثلاً برای یافتن همه‌ی برنامه‌هایی که نامشان شامل `zip` است:

```bash
ls /bin /usr/bin | sort | uniq | grep zip
```

```
bunzip2
bzip2
gunzip
gzip
unzip
zip
```

**گزینه‌های مفید `grep`:**
- `-i` — جستجوی بدون توجه به بزرگی/کوچکی حروف
- `-l` — فقط نام فایل‌های منطبق را نشان دهد
- `-v` — فقط خطوط **غیر**منطبق را نشان دهد
- `-w` — فقط کلمات کامل را پیدا کند

### `head` و `tail` — ابتدا و انتهای فایل

```bash
head -n 5 ls-output.txt    # 5 خط اول
tail -n 5 ls-output.txt    # 5 خط آخر
```

در لوله‌کشی:

```bash
ls /usr/bin | tail -n 5
```

`tail` همچنین می‌تواند به‌صورت زنده محتوای فایل را نشان دهد (مفید برای لاگ‌ها):

```bash
tail -f /var/log/messages
```

با Ctrl-c متوقف می‌شود.

### `tee` — خواندن و نوشتن همزمان

`tee` مثل یک سه‌راهی لوله عمل می‌کند — ورودی را هم به خروجی استاندارد و هم به یک فایل می‌فرستد:

```bash
ls /usr/bin | tee ls.txt | grep zip
```

```
bunzip2
bzip2
gunzip
gzip
...
```

## لینوکس یعنی تخیل

وقتی از من می‌پرسند تفاوت ویندوز و لینوکس چیست، از یک تشبیه اسباب‌بازی استفاده می‌کنم.

ویندوز مثل یک گیم‌بوی است — آن را می‌خری، روشن می‌کنی و بازی می‌کنی. اما بعد از مدتی خسته می‌شوی. می‌روی که بازی جدید بخری اما می‌گویند آنچه می‌خواهی وجود ندارد چون «تقاضای بازار ندارد». اسباب‌بازی‌ات محدود به بازی‌هایی است که دیگران برایت تصمیم گرفته‌اند.

لینوکس، از طرف دیگر، مثل بزرگ‌ترین مجموعه‌ی سازه‌های ساختمانی دنیاست. باز می‌کنی و مجموعه‌ای عظیم از قطعات می‌بینی. شروع می‌کنی بسازی — اول چیزهایی که پیشنهاد شده، بعد از ایده‌های خودت. هرگز نیازی به برگشتن به فروشگاه نداری. مجموعه شکل تخیل تو را می‌گیرد.

## خلاصه

همیشه مستندات هر دستوری که در این فصل یاد گرفتیم را بررسی کن. بیشترین استفاده‌ی پایه را دیده‌ایم، اما گزینه‌های جالب زیادی دارند. هرچه بیشتر با لینوکس کار کنی، می‌بینی که قابلیت تغییر مسیر برای حل مسائل خاص فوق‌العاده مفید است.
