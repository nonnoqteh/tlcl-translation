# فصل ۳ — کاوش در سیستم
### Chapter 3 – Exploring the System

---

حالا که می‌دانیم چطور در سیستم فایل حرکت کنیم، وقت گردش در سیستم لینوکس است. قبل از شروع، چند دستور جدید یاد می‌گیریم:

- `ls` — فهرست محتوای پوشه
- `file` — تعیین نوع فایل
- `less` — مشاهده‌ی محتوای فایل

---

## کار بیشتر با ls
### Having More Fun with ls

دستور `ls` احتمالاً پرکاربردترین دستور لینوکس است. با آن می‌توانیم محتوای پوشه را ببینیم و ویژگی‌های مهم فایل‌ها را بررسی کنیم:

```bash
ls
```

```
Desktop  Documents  Music  Pictures  Public  Templates  Videos
```

می‌توانیم محتوای هر پوشه‌ای را هم فهرست کنیم:

```bash
ls /usr
```

```
bin  games  include  lib  local  sbin  share  src
```

یا چند پوشه را با هم:

```bash
ls ~ /usr
```

```
/home/me:
Desktop  Documents  Music  Pictures  Public  Templates  Videos

/usr:
bin  games  include  lib  local  sbin  share  src
```

برای مشاهده‌ی جزئیات بیشتر از گزینه‌ی `-l` (فرمت طولانی) استفاده کنید:

```bash
ls -l
```

```
total 56
drwxrwxr-x 2 me me 4096 2017-10-26 17:20 Desktop
drwxrwxr-x 2 me me 4096 2017-10-26 17:20 Documents
drwxrwxr-x 2 me me 4096 2017-10-26 17:20 Music
...
```

---

## گزینه‌ها و آرگومان‌ها
### Options and Arguments

این ما را به نکته‌ی مهمی درباره‌ی نحوه‌ی کار دستورات می‌رساند. دستورات معمولاً با یک یا چند **گزینه** (option) دنبال می‌شوند که رفتارشان را تغییر می‌دهند، و بعد با یک یا چند **آرگومان** (argument). ساختار کلی این‌گونه است:

```
دستور -گزینه‌ها آرگومان‌ها
```

اکثر دستورات از گزینه‌های کوتاه با یک خط‌تیره استفاده می‌کنند (مثل `-l`). بسیاری هم، از جمله دستورات پروژه‌ی گنو، از گزینه‌های بلند با دو خط‌تیره پشتیبانی می‌کنند. همچنین می‌توان چند گزینه‌ی کوتاه را با هم ترکیب کرد:

```bash
ls -lt
```

این دستور با `-l` خروجی را به فرمت طولانی نشان می‌دهد و با `-t` بر اساس زمان آخرین تغییر مرتب می‌کند. با اضافه کردن `--reverse` ترتیب را معکوس می‌کنیم:

```bash
ls -lt --reverse
```

**جدول گزینه‌های رایج `ls`:**

| گزینه | گزینه‌ی بلند | توضیح |
|-------|-------------|-------|
| `-a` | `--all` | همه‌ی فایل‌ها را نشان می‌دهد، از جمله فایل‌های پنهان |
| `-A` | `--almost-all` | مثل `-a` ولی `.` و `..` را نشان نمی‌دهد |
| `-d` | `--directory` | خود پوشه را نشان می‌دهد، نه محتوایش |
| `-F` | `--classify` | یک نشانه به انتهای نام اضافه می‌کند (مثلاً `/` برای پوشه‌ها) |
| `-h` | `--human-readable` | اندازه‌ی فایل‌ها را به شکل خوانا نشان می‌دهد |
| `-l` | — | خروجی را به فرمت طولانی نشان می‌دهد |
| `-r` | `--reverse` | نتایج را معکوس نمایش می‌دهد |
| `-S` | — | بر اساس اندازه‌ی فایل مرتب می‌کند |
| `-t` | — | بر اساس زمان آخرین تغییر مرتب می‌کند |

## نگاه دقیق‌تر به فرمت طولانی
### A Longer Look at Long Format

گزینه‌ی `-l` خروجی جزئیاتی ارائه می‌دهد. در اینجا یک مثال واقعی از سیستم Ubuntu:

```
-rw-r--r-- 1 root root 3576296 2017-04-03 11:05 Experience ubuntu.ogg
-rw-r--r-- 1 root root 1186219 2017-04-03 11:05 kubuntu-leaflet.png
```

**معنی هر ستون:**

| ستون | معنی |
|------|------|
| `-rw-r--r--` | دسترسی‌ها (مجوزها). اولین کاراکتر نوع فایل، ۹ کاراکتر بعدی مجوزهای خواندن/نوشتن/اجرا |
| `1` | تعداد لینک‌های سخت (hard links) |
| `root` | نام مالک فایل |
| `root` | نام گروه مالک |
| `3576296` | اندازه‌ی فایل بر حسب بایت |
| `2017-04-03 11:05` | تاریخ و زمان آخرین تغییر |
| `Experience ubuntu.ogg` | نام فایل |

## تعیین نوع فایل با `file`
### Determining a File's Type with file

در لینوکس نام فایل لزوماً نوع محتوا را نشان نمی‌دهد. برای فهمیدن نوع واقعی یک فایل از دستور `file` استفاده کنید:

```bash
file picture.jpg
```

```
picture.jpg: JPEG image data, JFIF standard 1.01
```

یکی از اصول بنیادی سیستم‌های شبه‌یونیکس این است: **«همه چیز یک فایل است.»**

---

## مشاهده‌ی محتوای فایل با `less`
### Viewing File Contents with less

دستور `less` برنامه‌ای برای مشاهده‌ی فایل‌های متنی است. در سیستم لینوکس فایل‌های متنی زیادی وجود دارند که با `less` می‌توانیم آن‌ها را بررسی کنیم.

> **متن چیست؟** متن ساده (plain text) یک روش نمایش اطلاعات است که در آن کاراکترها به اعداد نگاشت می‌شوند. ساده‌ترین استاندارد آن ASCII است. برخلاف اسناد پردازشگر کلمه مثل Word، متن ساده فقط خود کاراکترها را دارد و هیچ اطلاعات قالب‌بندی‌ای ندارد.

استفاده از `less`:

```bash
less نام_فایل
```

مثلاً برای مشاهده‌ی فایل حساب‌های کاربری سیستم:

```bash
less /etc/passwd
```

**دستورات رایج در `less`:**

| دستور | عملکرد |
|-------|--------|
| `Page Up` یا `b` | یک صفحه به بالا |
| `Page Down` یا `Space` | یک صفحه به پایین |
| `↑` | یک خط به بالا |
| `↓` | یک خط به پایین |
| `G` | رفتن به انتهای فایل |
| `1G` یا `g` | رفتن به ابتدای فایل |
| `/کلمه` | جستجو به جلو |
| `n` | رفتن به نتیجه‌ی بعدی جستجو |
| `h` | نمایش راهنما |
| `q` | خروج |

> **کمتر بیشتر است:** نام `less` بازی با عبارت «less is more» است — شعار معماران مدرنیست. برنامه‌ی قدیمی‌تر `more` فقط می‌توانست به جلو برود، اما `less` امکان حرکت در هر دو جهت را دارد.

## گردش راهنمایی‌شده در سیستم
### Taking a Guided Tour

ساختار سیستم فایل لینوکس بر اساس استاندارد **Linux Filesystem Hierarchy Standard** طراحی شده. بیایید در سیستم فایل گردش کنیم:

1. با `cd` وارد یک پوشه شوید
2. با `ls -l` محتوایش را فهرست کنید
3. اگر فایل جالبی دیدید، با `file` نوعش را بررسی کنید
4. اگر به نظر می‌رسد متن باشد، با `less` محتوایش را ببینید
5. اگر تصادفاً یک فایل غیرمتنی را باز کردید و ترمینال به هم ریخت، دستور `reset` را تایپ کنید

**پوشه‌های مهم در لینوکس:**

| پوشه | توضیح |
|------|-------|
| `/` | پوشه‌ی ریشه — جایی که همه چیز از آنجا شروع می‌شود |
| `/bin` | برنامه‌های ضروری برای بوت و اجرای سیستم |
| `/boot` | هسته‌ی لینوکس و فایل‌های مربوط به بوت |
| `/dev` | پوشه‌ی دستگاه‌ها — «همه چیز یک فایل است» |
| `/etc` | فایل‌های تنظیمات سراسری سیستم |
| `/home` | پوشه‌های خانگی کاربران |
| `/lib` | کتابخانه‌های مشترک برنامه‌ها |
| `/media` | نقاط اتصال رسانه‌های قابل جابجایی (USB، CD) |
| `/opt` | نرم‌افزارهای «اختیاری» |
| `/proc` | سیستم فایل مجازی — پنجره‌ای به هسته‌ی لینوکس |
| `/root` | پوشه‌ی خانگی حساب root |
| `/sbin` | برنامه‌های مدیریت سیستم |
| `/tmp` | فایل‌های موقت |
| `/usr` | درخت بزرگ‌ترین پوشه — شامل همه‌ی برنامه‌ها و پشتیبانی‌ها |
| `/usr/bin` | برنامه‌های اجرایی نصب‌شده توسط توزیع |
| `/usr/share/doc` | مستندات بسته‌های نرم‌افزاری |
| `/var` | داده‌های متغیر (پایگاه داده، لاگ‌ها، ایمیل‌ها) |
| `/var/log` | فایل‌های لاگ سیستم |

## پیوندهای نمادین (Symbolic Links)
### Symbolic Links

در فهرست پوشه‌ها گاهی چنین ورودی‌هایی می‌بینیم:

```
lrwxrwxrwx 1 root root 11 2007-08-11 07:34 libc.so.6 -> libc-2.6.so
```

اولین کاراکتر `l` است و به نظر می‌رسد دو نام فایل دارد — این یک **پیوند نمادین** (symbolic link) است، که به آن «لینک نرم» (soft link) یا «سیم‌لینک» (symlink) هم می‌گویند.

پیوندهای نمادین برای حل مشکل نسخه‌بندی به وجود آمدند: فرض کنید برنامه‌ای به فایلی به نام `foo` نیاز دارد که نسخه‌هایش مکرراً تغییر می‌کند. با یک پیوند نمادین `foo` که به `foo-2.6` اشاره می‌کند، همه‌ی برنامه‌ها می‌توانند `foo` را پیدا کنند و ما هم می‌توانیم نسخه‌ی نصب‌شده را ببینیم.

---

## پیوندهای سخت (Hard Links)
### Hard Links

نوع دیگری از پیوند هم وجود دارد: **پیوند سخت** (hard link). پیوندهای سخت هم به فایل‌ها اجازه می‌دهند چند نام داشته باشند، اما به روش متفاوتی. در فصل بعد بیشتر توضیح می‌دهیم.

---

## خلاصه

در این گردش چیزهای زیادی یاد گرفتیم. یکی از نکات مهم این است که لینوکس چقدر باز و شفاف است — فایل‌های مهم زیادی به شکل متن ساده‌ی قابل خواندن هستند. برخلاف بسیاری از سیستم‌های بسته، لینوکس همه چیز را برای بررسی و مطالعه در دسترس می‌گذارد.
