# فصل ۴ — کار با فایل‌ها و پوشه‌ها
### Chapter 4 – Manipulating Files and Directories

---

حالا آماده‌ی کار واقعی هستیم! این فصل دستورات زیر را معرفی می‌کند:

- `cp` — کپی فایل‌ها و پوشه‌ها
- `mv` — جابجایی/تغییر نام فایل‌ها و پوشه‌ها
- `mkdir` — ساختن پوشه
- `rm` — حذف فایل‌ها و پوشه‌ها
- `ln` — ساختن لینک‌های سخت و نمادین

این پنج دستور از پرکاربردترین دستورات لینوکس هستند.

درست است که بعضی از این کارها با مدیر فایل گرافیکی راحت‌تر است، اما خط فرمان **قدرت و انعطاف** دارد. مثلاً چطور می‌توان همه‌ی فایل‌های HTML یک پوشه را به پوشه‌ی دیگری کپی کرد، به شرطی که در مقصد وجود نداشته باشند یا نسخه‌ی قدیمی‌تری داشته باشند؟ با مدیر فایل سخت است، اما با خط فرمان ساده:

```bash
cp -u *.html destination
```

## کاراکترهای جایگزین (Wildcards)
### Wildcards

قبل از شروع، باید از یک ویژگی مهم شل صحبت کنیم: **کاراکترهای جایگزین** (wildcards یا globbing). این کاراکترها به ما امکان می‌دهند گروهی از فایل‌ها را با الگو انتخاب کنیم.

**کاراکترهای جایگزین:**

| کاراکتر | معنی |
|---------|------|
| `*` | هر تعداد کاراکتر (از جمله صفر کاراکتر) |
| `?` | دقیقاً یک کاراکتر |
| `[کاراکترها]` | هر کاراکتری که عضو مجموعه باشد |
| `[!کاراکترها]` یا `[^کاراکترها]` | هر کاراکتری که عضو مجموعه نباشد |
| `[[:class:]]` | هر کاراکتری که عضو کلاس مشخص باشد |

**کلاس‌های رایج:**

| کلاس | معنی |
|------|------|
| `[:alnum:]` | هر کاراکتر حرفی-عددی |
| `[:alpha:]` | هر حرف |
| `[:digit:]` | هر عدد |
| `[:lower:]` | هر حرف کوچک |
| `[:upper:]` | هر حرف بزرگ |

**مثال‌های کاربردی:**

| الگو | فایل‌های منطبق |
|------|---------------|
| `*` | همه‌ی فایل‌ها |
| `g*` | هر فایلی که با `g` شروع شود |
| `b*.txt` | هر فایلی که با `b` شروع و به `.txt` ختم شود |
| `Data???` | هر فایلی که با `Data` شروع و دقیقاً ۳ کاراکتر دیگر داشته باشد |
| `[abc]*` | هر فایلی که با `a`، `b` یا `c` شروع شود |
| `[[:upper:]]*` | هر فایلی که با حرف بزرگ شروع شود |
| `[![:digit:]]*` | هر فایلی که با عدد شروع نشود |

## mkdir — ساختن پوشه
### mkdir – Create Directories

```bash
mkdir پوشه...
```

```bash

```

## cp — کپی فایل‌ها و پوشه‌ها
### cp – Copy Files and Directories

```bash
cp item1 item2          # کپی یک فایل/پوشه
cp item... پوشه         # کپی چند مورد به یک پوشه
```

**گزینه‌های مهم `cp`:**

| گزینه | گزینه‌ی بلند | معنی |
|-------|------------|------|
| `-a` | `--archive` | کپی با تمام ویژگی‌ها شامل مجوزها و مالکیت |
| `-i` | `--interactive` | قبل از بازنویسی فایل موجود، تأیید بخواهد |
| `-r` | `--recursive` | برای کپی پوشه‌ها ضروری است |
| `-u` | `--update` | فقط فایل‌هایی را کپی کند که در مقصد وجود ندارند یا قدیمی‌ترند |
| `-v` | `--verbose` | پیام‌های توضیحی نمایش دهد |

**مثال‌ها:**

```bash
cp file1 file2              # کپی file1 به file2
cp -i file1 file2           # کپی با تأیید قبل از بازنویسی
cp file1 file2 dir1         # کپی دو فایل به پوشه‌ی dir1
cp dir1/* dir2              # کپی همه‌ی فایل‌های dir1 به dir2
cp -r dir1 dir2             # کپی کامل پوشه‌ی dir1 به dir2
```

## mv — جابجایی و تغییر نام فایل‌ها
### mv – Move and Rename Files

دستور `mv` هم برای جابجایی و هم برای تغییر نام فایل‌ها استفاده می‌شود:

```bash
mv item1 item2          # جابجایی یا تغییر نام
mv item... پوشه         # جابجایی چند مورد به یک پوشه
```

**گزینه‌های مهم `mv`:**

| گزینه | گزینه‌ی بلند | معنی |
|-------|------------|------|
| `-i` | `--interactive` | قبل از بازنویسی تأیید بخواهد |
| `-u` | `--update` | فقط فایل‌های جدیدتر را جابجا کند |
| `-v` | `--verbose` | پیام‌های توضیحی نمایش دهد |

**مثال‌ها:**

```bash
mv file1 file2          # تغییر نام file1 به file2
mv -i file1 file2       # با تأیید قبل از بازنویسی
mv file1 file2 dir1     # انتقال دو فایل به dir1
mv dir1 dir2            # اگر dir2 نباشد، dir1 به dir2 تغییر نام می‌یابد
```

## rm — حذف فایل‌ها و پوشه‌ها
### rm – Remove Files and Directories

```bash
rm item...
```

**گزینه‌های مهم `rm`:**

| گزینه | گزینه‌ی بلند | معنی |
|-------|------------|------|
| `-i` | `--interactive` | قبل از حذف تأیید بخواهد |
| `-r` | `--recursive` | پوشه‌ها و زیرپوشه‌هایشان را حذف کند |
| `-f` | `--force` | بدون تأیید و بدون خطا برای فایل‌های ناموجود |
| `-v` | `--verbose` | پیام‌های توضیحی نمایش دهد |

**مثال‌ها:**

```bash
rm file1                # حذف بی‌سروصدا
rm -i file1             # حذف با تأیید
rm -r file1 dir1        # حذف فایل و پوشه (با زیرپوشه‌هایش)
rm -rf file1 dir1       # حذف بدون تأیید
```

> **⚠️ مراقب `rm` باشید!**
> سیستم‌های شبه‌یونیکس مثل لینوکس **دستور `undelete` ندارند.** هر چیزی که با `rm` حذف کنید، رفته. لینوکس فرض می‌کند که می‌دانید چه می‌کنید.
>
> مخصوصاً با کاراکترهای جایگزین مراقب باشید. قبل از `rm`، ابتدا با `ls` آزمایش کنید تا ببینید کدام فایل‌ها انتخاب می‌شوند. مثلاً اگر می‌خواهید فایل‌های `.html` را حذف کنید:
> ```bash
> rm *.html    # درست
> rm * .html   # خطرناک! با فاصله‌ی اضافی همه فایل‌ها حذف می‌شوند!
> ```

## ln — ساختن لینک
### ln – Create Links

```bash
ln file link            # لینک سخت
ln -s item link         # لینک نمادین
```

### لینک‌های سخت (Hard Links)

لینک‌های سخت روش اصلی یونیکس برای ایجاد لینک است. هر فایل حداقل یک لینک سخت دارد که نامش را می‌دهد. محدودیت‌های لینک سخت:

1. نمی‌تواند به فایلی در پارتیشن دیگری اشاره کند
2. نمی‌تواند به یک پوشه اشاره کند

### لینک‌های نمادین (Symbolic Links)

لینک‌های نمادین برای غلبه بر محدودیت‌های لینک سخت ساخته شدند. مثل میانبرهای ویندوز عمل می‌کنند — فایلی مخصوص است که یک اشاره‌گر متنی به فایل یا پوشه‌ی هدف دارد.

## بیایید یک زمین‌بازی بسازیم!
### Let's Build a Playground

بیایید همه‌ی این دستورات را در عمل امتحان کنیم:

```bash
cd
mkdir playground
cd playground
mkdir dir1 dir2
```

**کپی فایل‌ها:**

```bash
cp /etc/passwd .
ls -l
```

```
drwxrwxr-x 2 me me 4096 2025-01-10 dir1
drwxrwxr-x 2 me me 4096 2025-01-10 dir2
-rw-r--r-- 1 me me 1650 2025-01-10 passwd
```

```bash
cp -v /etc/passwd .
```

```
'/etc/passwd' -> './passwd'
```

```bash
cp -i /etc/passwd .
```

```
cp: overwrite './passwd'?
```

**جابجایی و تغییر نام:**

```bash
mv passwd fun
mv fun dir1
mv dir1/fun dir2
mv dir2/fun .
```

**ساختن لینک‌های سخت:**

```bash
ln fun fun-hard
ln fun dir1/fun-hard
ln fun dir2/fun-hard
ls -l
```

```
-rw-r--r-- 4 me me 1650 2025-01-10 fun
-rw-r--r-- 4 me me 1650 2025-01-10 fun-hard
```

توجه کنید عدد `4` در ستون دوم — تعداد لینک‌های سخت است. برای دیدن شماره‌ی inode (که نشان می‌دهد fun و fun-hard یک فایل هستند):

```bash
ls -li
```

```
12353538 -rw-r--r-- 4 me me 1650 fun
12353538 -rw-r--r-- 4 me me 1650 fun-hard
```

هر دو شماره‌ی inode یکسانی دارند — واقعاً یک فایل هستند.

**ساختن لینک‌های نمادین:**

```bash
ln -s fun fun-sym
ln -s ../fun dir1/fun-sym
ln -s ../fun dir2/fun-sym
ls -l dir1
```

```
-rw-r--r-- 4 me me 1650 fun-hard
lrwxrwxrwx 1 me me    6 fun-sym -> ../fun
```

**پاک‌سازی:**

```bash
rm fun-hard
rm -i fun
ls -l
```

پس از حذف `fun`، لینک `fun-sym` **شکسته** می‌شود (به فایلی اشاره می‌کند که وجود ندارد).

```bash
rm fun-sym dir1-sym
cd
rm -r playground
```

> **ساختن سیم‌لینک در محیط گرافیکی:** در GNOME با نگه‌داشتن Ctrl+Shift و کشیدن فایل، لینک نمادین می‌سازد. در KDE هنگام رها کردن فایل منویی ظاهر می‌شود.
