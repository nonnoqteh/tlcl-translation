# فصل ۲ — ناوبری
### Chapter 2 – Navigation

---

اولین چیزی که باید یاد بگیریم (غیر از تایپ کردن!) ناوبری در سیستم فایل لینوکس است. در این فصل این دستورها را معرفی می‌کنیم:

- `pwd` — نمایش پوشه‌ی جاری
- `cd` — تغییر پوشه
- `ls` — فهرست کردن محتوای پوشه

---

## درک درخت سیستم فایل
### Understanding the File System Tree

مثل ویندوز، سیستم‌عامل‌های شبه‌یونیکس مثل لینوکس فایل‌ها را در یک **ساختار پوشه‌ای سلسله‌مراتبی** (hierarchical directory structure) سازماندهی می‌کنند — شبیه به یک درخت. اولین پوشه در سیستم فایل، **پوشه‌ی ریشه** (root directory) نام دارد که با `/` نشان داده می‌شود.

برخلاف ویندوز که هر دستگاه ذخیره‌سازی درخت فایل جداگانه‌ای دارد، لینوکس همیشه یک درخت فایل واحد دارد، صرف‌نظر از اینکه چند دیسک یا دستگاه به رایانه متصل باشد.

---

## پوشه‌ی جاری
### The Current Working Directory

تصور کنید سیستم فایل یک هزارتوی درخت‌مانند است و ما در وسط آن ایستاده‌ایم. در هر لحظه داخل یک پوشه هستیم که **پوشه‌ی جاری** (current working directory) نامیده می‌شود. برای نمایش آن از دستور `pwd` (print working directory) استفاده کنید:

```bash
pwd
```

```
/home/me
```

هنگام ورود به سیستم (یا شروع نشست ترمینال)، پوشه‌ی جاری روی **پوشه‌ی خانگی** (home directory) تنظیم می‌شود. هر حساب کاربری یک پوشه‌ی خانگی دارد که تنها جایی است که کاربر عادی مجاز به نوشتن فایل است.

---

## فهرست کردن محتوای پوشه
### Listing the Contents of a Directory

برای فهرست کردن فایل‌ها و پوشه‌های پوشه‌ی جاری از دستور `ls` استفاده کنید:

```bash
ls
```

```
Desktop  Documents  Music  Pictures  Public  Templates  Videos
```

می‌توانیم محتوای هر پوشه‌ای را هم فهرست کنیم، نه فقط پوشه‌ی جاری. در فصل بعد بیشتر با `ls` کار می‌کنیم.

---

## تغییر پوشه‌ی جاری
### Changing the Current Working Directory

برای تغییر پوشه از دستور `cd` به همراه **مسیر** (pathname) پوشه‌ی مقصد استفاده کنید. مسیر را می‌توان به دو شکل نوشت: **مسیر مطلق** یا **مسیر نسبی**.

### مسیر مطلق
### Absolute Pathnames

مسیر مطلق از پوشه‌ی ریشه شروع می‌شود و شاخه به شاخه تا مقصد ادامه می‌یابد. مثلاً اکثر برنامه‌های سیستم در `/usr/bin` هستند:

```bash
cd /usr/bin
pwd
```

```
/usr/bin
```

```bash
ls
```

```
...فهرستی از فایل‌های بسیار زیاد...
```

توجه کنید که فراخوان شل هم تغییر کرد تا نام پوشه‌ی جاری را نشان دهد.

### مسیر نسبی
### Relative Pathnames

مسیر نسبی از پوشه‌ی جاری شروع می‌شود. دو نشانه‌ی خاص داریم:
- `.` (نقطه) — به پوشه‌ی جاری اشاره دارد
- `..` (نقطه‌نقطه) — به پوشه‌ی والد اشاره دارد

مثال: تغییر به پوشه‌ی والد `/usr/bin` که `/usr` است:

```bash
cd /usr/bin
cd ..
pwd
```

```
/usr
```

یا با مسیر مطلق:

```bash
cd /usr
pwd
```

```
/usr
```

هر دو روش نتیجه‌ی یکسانی دارند. کدام را انتخاب کنیم؟ **آن که تایپ کمتری نیاز دارد!**

در اکثر موارد می‌توانیم `./` ابتدای مسیر نسبی را حذف کنیم — ضمنی در نظر گرفته می‌شود:

```bash
cd bin
```

معادل `cd ./bin` است.

## میانبرهای مفید
### Some Helpful Shortcuts

| میانبر | نتیجه |
|--------|-------|
| `cd` | رفتن به پوشه‌ی خانگی |
| `cd -` | رفتن به پوشه‌ی قبلی |
| `cd ~نام_کاربر` | رفتن به پوشه‌ی خانگی آن کاربر (مثلاً `cd ~bob`) |

## نکات مهم درباره‌ی نام فایل‌ها
### Important Facts About Filenames

۱. **فایل‌هایی که نامشان با نقطه شروع می‌شود پنهان هستند.** `ls` آن‌ها را نشان نمی‌دهد مگر با گزینه‌ی `-a`.

۲. **نام فایل‌ها و دستورات در لینوکس حساس به حروف بزرگ و کوچک هستند.** فایل‌های `File1` و `file1` دو فایل متفاوتند.

۳. **لینوکس مفهوم «پسوند فایل» ندارد.** می‌توانید هر نامی به فایل بدهید. نوع و هدف فایل با روش‌های دیگر تعیین می‌شود.

۴. **از فاصله در نام فایل پرهیز کنید.** اگر می‌خواهید فاصله را نشان دهید، از خط‌زیر (`_`) استفاده کنید. بعداً از خودتان تشکر خواهید کرد!
