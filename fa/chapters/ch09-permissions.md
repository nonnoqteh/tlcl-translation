# فصل ۹ — مجوزها
### Chapter 9 – Permissions

---

سیستم‌های عامل در سنت یونیکس با سیستم‌هایی مثل MS-DOS این تفاوت را دارند: نه تنها **چندکاره** (multitasking) بلکه **چندکاربره** (multi-user) هستند.

چند کاربر می‌توانند همزمان از یک رایانه استفاده کنند — مثلاً از راه دور از طریق `ssh`. این قابلیت نیازمند یک مکانیزم محافظت از کاربران در برابر یکدیگر است. در این فصل این بخش اساسی امنیت سیستم را بررسی می‌کنیم.

دستورات این فصل:
- `id` — نمایش هویت کاربر
- `chmod` — تغییر مجوز فایل
- `umask` — تنظیم مجوز پیش‌فرض
- `su` — اجرای شل به عنوان کاربر دیگر
- `sudo` — اجرای دستور به عنوان کاربر دیگر
- `chown` — تغییر مالک فایل
- `chgrp` — تغییر گروه مالک فایل
- `passwd` — تغییر رمز عبور

---

## کاربران، اعضای گروه و بقیه
### Users, Group Members, and Everybody Else

وقتی در فصل ۳ سیستم را بررسی کردیم، شاید با این خطا مواجه شدیم:

```bash
file /etc/shadow
```

```
/etc/shadow: regular file, no read permission
```

```bash
less /etc/shadow
```

```
/etc/shadow: Permission denied
```

دلیل: ما به عنوان کاربر عادی مجاز به خواندن این فایل نیستیم.

در مدل امنیتی یونیکس:
- **کاربر** می‌تواند مالک فایل‌ها و پوشه‌ها باشد
- **گروه** مجموعه‌ای از یک یا چند کاربر است که مالک می‌تواند به آن‌ها دسترسی بدهد
- **دیگران** (others / world) بقیه‌ی کاربران

برای مشاهده‌ی هویت‌مان:

```bash
id
```

```
uid=500(me) gid=500(me) groups=500(me)
```

یا در Ubuntu:

```bash
uid=1000(me) gid=1000(me) groups=4(adm),20(dialout),24(cdrom),...
```

این اطلاعات از فایل‌های متنی می‌آیند:
- `/etc/passwd` — حساب‌های کاربری
- `/etc/group` — گروه‌ها
- `/etc/shadow` — رمزهای عبور (رمزگذاری‌شده)

## خواندن، نوشتن و اجرا
### Reading, Writing, and Executing

دسترسی‌های فایل و پوشه در سه سطح تعریف می‌شوند: **خواندن** (r)، **نوشتن** (w) و **اجرا** (x).

```bash
ls -l foo.txt
```

```
-rw-rw-r-- 1 me me 0 2016-03-06 14:52 foo.txt
```

۱۰ کاراکتر اول **ویژگی‌های فایل** هستند:

- کاراکتر اول: نوع فایل
  - `-` فایل عادی
  - `d` پوشه
  - `l` لینک نمادین
  - `c` دستگاه کاراکتری
  - `b` دستگاه بلاکی

- ۹ کاراکتر بعدی: **حالت فایل** (file mode)

```
مالک    گروه    بقیه
rwx     rwx     rwx
```

**معنی مجوزها:**

| مجوز | فایل | پوشه |
|------|------|------|
| `r` | اجازه‌ی باز کردن و خواندن | اجازه‌ی فهرست کردن محتوا |
| `w` | اجازه‌ی نوشتن و کوتاه کردن | اجازه‌ی ساختن، حذف و تغییر نام فایل‌ها |
| `x` | اجازه‌ی اجرا به عنوان برنامه | اجازه‌ی ورود با `cd` |

**مثال‌های رایج:**

| مجوز | معنی |
|------|------|
| `-rwx------` | فقط مالک می‌تواند بخواند، بنویسد و اجرا کند |
| `-rw-------` | فقط مالک می‌تواند بخواند و بنویسد |
| `-rw-r--r--` | مالک می‌خواند/می‌نویسد، گروه و بقیه فقط می‌خوانند |
| `-rwxr-xr-x` | مالک همه‌کاره است، بقیه می‌توانند بخوانند و اجرا کنند |
| `drwxrwx---` | مالک و گروه می‌توانند وارد پوشه شوند و فایل بسازند/حذف کنند |

## `chmod` — تغییر مجوز
### chmod – Change File Mode

تنها مالک فایل یا سوپریوزر می‌توانند مجوز تغییر دهند. `chmod` دو روش دارد:

### روش اکتال (Octal)

هر رقم اکتال ۳ بیت را نمایش می‌دهد:

| اکتال | باینری | مجوز |
|-------|--------|------|
| 0 | 000 | `---` |
| 1 | 001 | `--x` |
| 2 | 010 | `-w-` |
| 3 | 011 | `-wx` |
| 4 | 100 | `r--` |
| 5 | 101 | `r-x` |
| 6 | 110 | `rw-` |
| 7 | 111 | `rwx` |

```bash
chmod 600 foo.txt
ls -l foo.txt
```

```
-rw------- 1 me me 0 2016-03-06 14:52 foo.txt
```

مقادیر رایج: **7** (rwx)، **6** (rw-)، **5** (r-x)، **4** (r--)، **0** (---)

### روش نمادین (Symbolic)

```
chmod [ugoa][+-=][rwx] فایل
```

| نماد | معنی |
|------|------|
| `u` | مالک (user) |
| `g` | گروه (group) |
| `o` | بقیه (others) |
| `a` | همه (all) |

| عملگر | معنی |
|-------|------|
| `+` | افزودن مجوز |
| `-` | حذف مجوز |
| `=` | تنظیم دقیق مجوز |

مثال‌ها:

```bash
chmod u+x file          # افزودن مجوز اجرا برای مالک
chmod u-x file          # حذف مجوز اجرا از مالک
chmod +x file           # افزودن مجوز اجرا برای همه
chmod go=rw file        # تنظیم خواندن/نوشتن برای گروه و بقیه
chmod u+x,go=rx file    # چند تغییر با کاما جدا می‌شوند
```

## `umask` — تنظیم مجوز پیش‌فرض
### umask – Set Default Permissions

`umask` مجوز پیش‌فرض فایل‌های جدید را کنترل می‌کند. یک **ماسک** است که بیت‌هایی را از مجوز حذف می‌کند:

```bash
umask
```

```
0002
```

```bash
> foo.txt
ls -l foo.txt
```

```
-rw-rw-r-- 1 me me 0 foo.txt
```

توضیح: مجوز پایه `rw-rw-rw-` است. umask 0002 بیت نوشتن «بقیه» را حذف می‌کند.

## مجوزهای ویژه
### Some Special Permissions

علاوه بر r، w، x سه مجوز خاص دیگر هم وجود دارند:

- **setuid** (اکتال 4000): برنامه با مجوزهای مالکش اجرا می‌شود نه کاربر جاری. `chmod u+s program`
- **setgid** (اکتال 2000): فایل‌های جدید در پوشه گروه پوشه را به ارث می‌برند. `chmod g+s dir`
- **sticky bit** (اکتال 1000): فایل‌ها را فقط مالکشان یا سوپریوزر می‌توانند حذف کنند (مثل `/tmp`). `chmod +t dir`

## تغییر هویت
### Changing Identities

سه روش برای به‌عهده گرفتن هویت کاربر دیگر:

### `su` — اجرای شل به عنوان کاربر دیگر

```bash
su [-[l]] [کاربر]
```

برای تبدیل شدن به سوپریوزر:

```bash
su -
```

```
Password:
[root@linuxbox ~]#
```

با `exit` برمی‌گردیم. برای اجرای یک دستور به عنوان کاربر دیگر:

```bash
su -c 'ls -l /root/*'
```

### `sudo` — اجرای دستور به عنوان کاربر دیگر

`sudo` امکانات بیشتری دارد:
- مدیر می‌تواند مجوزهای دقیق تعریف کند
- به رمز عبور سوپریوزر نیاز نیست — رمز کاربر جاری کافی است
- شل جدید باز نمی‌شود

```bash
sudo backup_script
```

برای دیدن مجوزهای اعطاشده:

```bash
sudo -l
```

## `chown` — تغییر مالک
### chown – Change File Owner and Group

```bash
chown [مالک][:[گروه]] فایل...
```

مثال‌ها:

| آرگومان | نتیجه |
|---------|-------|
| `bob` | مالکیت به bob منتقل می‌شود |
| `bob:users` | مالکیت به bob و گروه به users |
| `:admins` | فقط گروه به admins تغییر می‌کند |
| `bob:` | مالک به bob، گروه به گروه ورود bob |

**مثال عملی:** Janet می‌خواهد فایلی را برای tony کپی کند:

```bash
sudo cp myfile.txt ~tony
sudo chown tony: ~tony/myfile.txt
```

## `chgrp` — تغییر گروه
### chgrp – Change Group Ownership

مشابه `chown` اما فقط گروه را تغییر می‌دهد.

## تمرین عملی: پوشه‌ی اشتراکی

janet و tony می‌خواهند یک پوشه‌ی موسیقی مشترک داشته باشند:

```bash
# ساختن گروه
sudo groupadd music
sudo usermod -a -G music janet
sudo usermod -a -G music tony

# ساختن پوشه
sudo mkdir /usr/local/share/Music
sudo chown :music /usr/local/share/Music
sudo chmod 2775 /usr/local/share/Music
ls -ld /usr/local/share/Music
```

```
drwxrwsr-x 2 root music 4096 2025-03-21 18:05 /usr/local/share/Music
```

حالا هر دو می‌توانند فایل بسازند. اما برای اینکه فایل‌های ساخته‌شده توسط هرکدام توسط دیگری هم قابل ویرایش باشند، umask را تنظیم می‌کنیم:

```bash
umask 0002
```

## تغییر رمز عبور
### Changing Your Password

```bash
passwd
```

```
(current) UNIX password:
New UNIX password:
```

`passwd` رمزهای ضعیف (کوتاه، شبیه به قبلی، کلمه‌ی دیکشنری) را رد می‌کند.

## خلاصه

در این فصل دیدیم که سیستم‌های شبه‌یونیکس چطور مجوزهای کاربران را برای دسترسی به فایل‌ها و پوشه‌ها مدیریت می‌کنند. این ایده‌های بنیادین از روزهای اولیه‌ی یونیکس می‌آیند و آزمون زمان را پس داده‌اند.
