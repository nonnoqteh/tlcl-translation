# فصل ۱۱ — محیط
### Chapter 11 – The Environment

---

همان‌طور که قبلاً دیدیم، شل در طول نشست (session) اطلاعاتی را نگه می‌دارد که **محیط** (environment) نامیده می‌شود. برنامه‌ها از این محیط برای تنظیم رفتارشان استفاده می‌کنند. اکثر برنامه‌ها از **فایل‌های پیکربندی** (configuration files) هم استفاده می‌کنند.

دستورات این فصل:
- `printenv` — چاپ بخشی یا همه‌ی محیط
- `set` — تنظیم گزینه‌های شل
- `export` — صادر کردن متغیر به محیط
- `alias` — ساختن نام مستعار برای دستور

---

## محیط چه چیزی است؟
### What Is Stored in the Environment?

شل اطلاعات را در دو نوع متغیر ذخیره می‌کند:

- **متغیرهای محیطی** (environment variables) — اطلاعاتی که از فرآیند والد به فرزند به ارث می‌رسند
- **متغیرهای شل** (shell variables) — اطلاعاتی که فقط در نشست شل جاری هستند

علاوه بر متغیرها، شل داده‌های برنامه‌نویسی مثل **توابع** و **نام‌های مستعار** هم ذخیره می‌کند.

### بررسی محیط

برای مشاهده‌ی متغیرهای محیطی:

```bash
printenv | less
```

برای مشاهده‌ی یک متغیر خاص:

```bash
printenv USER
```

```
me
```

با `set` هم می‌توان هم متغیرهای شل و هم محیطی را دید:

```bash
set | less
```

برای مشاهده‌ی محتوای هر متغیر با `echo`:

```bash
echo $HOME
```

```
/home/me
```

**متغیرهای محیطی مهم:**

| متغیر | محتوا |
|-------|-------|
| `DISPLAY` | نام نمایشگر گرافیکی (در سیستم‌های X11) |
| `EDITOR` | ویرایشگر پیش‌فرض |
| `HOME` | مسیر پوشه‌ی خانگی |
| `LANG` | کاراکترست و ترتیب مرتب‌سازی |
| `OLDPWD` | پوشه‌ی کاری قبلی |
| `PAGER` | نام برنامه‌ی صفحه‌بندی پیش‌فرض (معمولاً `less`) |
| `PATH` | فهرست پوشه‌های جستجو برای اجرایی‌ها |
| `PS1` | رشته‌ی اول پرامپت — تعریف می‌کند پرامپت چه شکلی باشد |
| `PWD` | پوشه‌ی کاری جاری |
| `SHELL` | نام شل |
| `TERM` | نوع ترمینال |
| `TZ` | منطقه‌ی زمانی |
| `USER` | نام کاربری |

## محیط چگونه ایجاد می‌شود؟
### How Is the Environment Established?

وقتی وارد سیستم می‌شوی، فرآیند bash یک سری فایل‌های پیکربندی به نام **فایل‌های راه‌اندازی** (startup files) را می‌خواند. این فایل‌ها محیط پیش‌فرض را تعریف می‌کنند.

دقیقاً کدام فایل‌ها خوانده می‌شوند بستگی دارد به نوع نشست:

### نشست ورود (Login Shell Session)

| فایل | محتوا |
|------|-------|
| `/etc/profile` | اسکریپت پیکربندی سراسری برای همه‌ی کاربران |
| `~/.bash_profile` | فایل راه‌اندازی شخصی |
| `~/.bash_login` | اگر `~/.bash_profile` نباشد، این خوانده می‌شود |
| `~/.profile` | اگر هیچ‌کدام نباشند، این خوانده می‌شود |

### نشست غیر ورود (Non-Login Shell Session)

| فایل | محتوا |
|------|-------|
| `/etc/bash.bashrc` | اسکریپت پیکربندی سراسری |
| `~/.bashrc` | فایل راه‌اندازی شخصی |

معمولاً `~/.bash_profile` طوری نوشته شده که `~/.bashrc` را هم صدا می‌زند:

```bash
# از ~/.bash_profile
if [ -f ~/.bashrc ]; then
    . ~/.bashrc
fi
```

---

## محتوای فایل راه‌اندازی
### What's in a Startup File?

یک `~/.bashrc` معمولی:

```bash
# Source global definitions
if [ -f /etc/bashrc ]; then
    . /etc/bashrc
fi

# User specific aliases and functions
alias ll='ls -l --color=auto'
alias la='ls -la --color=auto'
alias l.='ls -d .* --color=auto'

# User specific environment
PATH=$PATH:$HOME/bin
export PATH
```

خطوطی که با `#` شروع می‌شوند **توضیح** (comment) هستند و اجرا نمی‌شوند.

## تغییر محیط
### Modifying the Environment

**قانون کلی:** برای افزودن پوشه به `PATH` یا تعریف متغیرهای اضافه، تغییر را در `~/.bash_profile` (یا معادلش) بده. برای بقیه، `~/.bashrc` مناسب‌تر است.

### ویرایش فایل‌های راه‌اندازی

**مراقب باش!** قبل از ویرایش، یک نسخه‌ی پشتیبان بساز:

```bash
cp ~/.bashrc ~/.bashrc.bak
```

حالا با ویرایشگر باز کن:

```bash
nano ~/.bashrc
```

یا:

```bash
gedit ~/.bashrc
```

### مثال: اضافه کردن یک پوشه به PATH

```bash
export PATH=$PATH:$HOME/bin
```

این خط پوشه‌ی `~/bin` را به `PATH` اضافه می‌کند — پوشه‌ای که می‌توانیم اسکریپت‌های شخصی‌مان را در آن بگذاریم.

### فعال‌سازی تغییرات

بعد از ویرایش، برای اعمال تغییرات بدون نیاز به خروج و ورود دوباره:

```bash
source ~/.bashrc
```

یا به‌صورت مختصر:

```bash
. ~/.bashrc
```

## خلاصه

درک محیط شل برای سفارشی‌سازی تجربه‌ی خط فرمان ضروری است. با `~/.bashrc` و `~/.bash_profile` می‌توانیم نام‌های مستعار، متغیرها و توابع دائمی بسازیم که در هر نشست در دسترس هستند.
